Problem 2: Null is a canonical JavaScript type, but it has some unusual or unexpected typeof() behavior. In 1-2 sentences, explain the unusual behavior that it has.
Null is a canonical JavaScript type and represents the absence and lack of identification for an object value. Given that "null" is retrieved in place of an absent object, it is unexpectedly recognized as an "object" by the typeof() function. Null is different from undefined, another canonical type.

Problem 3: First, in 1-2 sentences, please define type coercion.
Type coercion in JavaScript is the automatic conversion of data types to ensure program stability when executing operations. It involves adjusting the type of one element within an input and aligning it more harmoniously with another element in the same input. This process aims to facilitate the generation of an output, preserving program execution rather than halting with an error. This can lead to unexpected behavior and can sometimes cause "errors" within a program.
Examples:

  • console.log("3"+ 4 + 5 + "9"); returns "3459" in the console as a result of type coercion which is a rather unexpected output. Given that the input contains a mix of strings and ints, JavaScript changes all off the ints to strings in order to avoid crashing the program or throwing an error. This differs from Python where you are unable to concatenate strings and ints and would thus recieve a TypeError.
  • console.log([1, 2, 3]+5); returns "1,2,35" in the console as a result of type coercion which is also unexpected. Given that the input is attempting to concatenate an int to a list, JavaScript takes a shortcut by combining the int 5 with the last element in the list which is 3 in order to avoid throwing an error. In Python, inputing this would throw a TypeError given that you can only concatenate lists with other lists.
